### Deviations / decisions discovered during execution (not in the original plan)

- **Avo resolved to v4.x**, not 3.x as the original plan assumed (no version
  was pinned in the Gemfile). Avo 4's configuration/authorization syntax may
  differ from what's documented in the plan — **revisit in Phase 8**.
- Avo automatically generates a resource (`app/avo/resources/*.rb`) and a
  controller (`app/controllers/avo/*_controller.rb`) for every model created
  via `rails generate model`, even though the Avo admin panel itself hasn't
  been mounted yet. This is convenient (it gets a head start on Phase 8), but
  it means these files already exist and will keep growing with every new
  model — worth reviewing/organizing once we reach Phase 8.
- Docker bind mount on Windows/Git Bash: `-v "$(pwd):/app"` **does not work**
  (it silently creates an anonymous volume instead of mounting the host
  directory) — the Windows-style path must be used instead
  (`-v "D:/Sites:/app"`) together with `MSYS_NO_PATHCONV=1`. Already handled
  in every command used so far; documented here to avoid hitting it again.
- Git identity was not configured in this environment (neither locally nor
  globally) — configured locally for this repo only (see above).

### How to run / verify

```bash
# bring up the full stack
docker compose up -d

# prepare the dev database
docker compose run --rm web bin/rails db:prepare
docker compose run --rm web bin/rails db:seed

# prepare the test database and run specs
docker compose run --rm -e RAILS_ENV=test web bin/rails db:prepare
docker compose run --rm -e RAILS_ENV=test web bundle exec rspec

# visit http://localhost:3000 — log in with admin@example.com / changeme123
# (or your own ADMIN_EMAIL/ADMIN_PASSWORD, if customized in .env)
```

Verified end-to-end during this phase: `docker compose up` boots
`web`+`worker`+`db`+`redis` with no errors; admin login through the Devise
form works (POST `/users/sign_in` → redirect → authenticated home page
showing name and roles); RSpec suite has 14 examples, 0 failures.
